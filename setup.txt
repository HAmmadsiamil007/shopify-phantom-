1. THE GLOBAL CONFIG FILE
================================================================================

All configuration is in ONE file:

  C:\Users\hamma\.config\opencode\opencode.jsonc

This file is read by opencode on EVERY session start, regardless of which
project folder you open. It contains THREE sections:

  a) "plugin"  — Plugins to load (5 plugins)
  b) "skills"  — Paths to scan for skills (2 paths)
  c) "mcp"     — MCP servers to spawn (8 servers)

  NOTE: There is NO project-level config needed. The global file is enough.
  The file config/mcp-servers.json in this project is NOT used — it exists
  for reference only. The GLOBAL config takes precedence.

3. SKILLS — INSTALLATION & GLOBAL SETUP
================================================================================

Skills are NOT npm packages. They are directories containing a SKILL.md file
that provides instructions to the AI agent.

3a. Installation
  -----------------------------------------
  Skills are placed in:
    GLOBAL:  C:\Users\hamma\.agents\skills\  (119 skills)
    PROJECT: <project>\.agents\skills\       (optional per-project copy)

  Skills come from various sources:
  - anthopic/skills (Claude API, PDF, DOCX, etc.)
  - vercel-labs/agent-skills (React, design, writing)
  - vercel-labs/skills (skill discovery)
  - Custom/manual installs (Shopify skills, Stitch skills, etc.)

  Installation command (if adding new skills):
    npx skills add <source> --skill <name> -y

3b. How Skills are Made Global
  -----------------------------------------
  In opencode.jsonc:
    "skills": {
      "paths": [
        "~/.agents/skills",     // Global — shared across ALL projects
        ".agents/skills"         // Project-specific override
      ]
    }

  The first path (~/.agents/skills/) is the GLOBAL one. Any skill folder
  placed here is automatically available in every project.

  The second path (.agents/skills/) is project-relative. If a project has
  its own skills folder, those are loaded too (in addition to global).

3c. Skill Loading
  -----------------------------------------
  Each project also has .opencode/skill-loader-config.json which lists
  all 119 skills under "alwaysLoad". This tells the skill tool which
  skills to make available.

  To load a skill in-session:
    skill load <skill-name>

3d. Adding a New Skill (Making It Global)
  -----------------------------------------
  1. Create directory: mkdir ~\.agents\skills\my-skill
  2. Create SKILL.md:  write instructions in markdown
  3. Done! — It's now available in ALL future sessions.
     (No config changes needed — opencode scans the paths automatically)

  Optional: Add to .opencode/skill-loader-config.json if you want it in
  the alwaysLoad list.

    npm install opencode-quota --prefix ~\.config\opencode

  opencode-snippets (latest) — Code snippet management
    npm install opencode-snippets --prefix ~\.config\opencode

  superpowers v6.1.1 — Skills runtime + workflow tools (obra/superpowers)
    npm install https://github.com/obra/superpowers/tarball/main `
      --prefix ~\.config\opencode
    Note: Installed via tarball URL (bypasses git dependency)

4b. How Plugins Are Made Global
  -----------------------------------------
  In opencode.jsonc:
    "plugin": [
      "~/.config/opencode/node_modules/@opencode-ai/plugin",
      "~/.config/opencode/node_modules/opencode-mem",
      "~/.config/opencode/node_modules/opencode-quota",
      "~/.config/opencode/node_modules/opencode-snippets",
      "~/.config/opencode/node_modules/superpowers"
    ]

  Each entry is a path to the installed npm package. opencode loads them
  at startup in order.

4c. Adding a New Plugin (Making It Global)
  -----------------------------------------
  1. Install: npm install <package> --prefix ~\.config\opencode
  2. Add to "plugin" array in opencode.jsonc
  3. Restart opencode — plugin loads automatically

5. HOW EVERYTHING WORKS AS GLOBAL DEFAULT
================================================================================

The magic is in ONE file: C:\Users\hamma\.config\opencode\opencode.jsonc

When you open ANY folder in opencode:
  1. opencode reads the global config file
  2. It spawns all 8 MCP servers defined in "mcp" section
  3. It loads all 5 plugins from the "plugin" array
  4. It scans ~/.agents/skills/ and .agents/skills/ for skills
  5. All skills, MCP tools, and plugin features become available

There is NO need to:
  - Copy config files between projects
  - Re-install MCP servers per project
  - Re-install plugins per project
  - Set up skills per project

One-time setup → Works everywhere.

7. TROUBLESHOOTING
================================================================================

7a. MCP not connecting
  - Check opencode.jsonc has the server in "mcp" with "enabled": true
  - Check the binary path exists (npm root -g to find global node_modules)
  - Try running the command manually in terminal
  - For open-design-mcp: start the daemon first (od command)

7b. Skill not loading
  - Check ~/.agents/skills/<name>/SKILL.md exists
  - Check .opencode/skill-loader-config.json has the skill in alwaysLoad
  - Try: skill load <name>

7c. Plugin not loading
  - Check the directory exists at the path in "plugin" array
  - Check opencode.jsonc syntax (valid JSON)
  - Try: npm install <plugin> --prefix ~\.config\opencode

7d. mcp-git failing
  - Install Git for Windows from git-scm.com
  - Add to PATH: $env:Path += ";C:\Program Files\Git\bin"
  - Restart opencode

7e. open-design-mcp failing
  - Start daemon: Start-Job -Name od-daemon -ScriptBlock { od }
  - Check OD_DAEMON_URL matches: http://127.0.0.1:7456

7f. serena not activated
  - Call: serena_activate_project(project: "path/to/project")
  - Or: serena project init (from terminal)
